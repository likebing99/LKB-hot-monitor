# AI 分析 Prompt 模板

本文档定义了 Agent 对搜索结果做分析时使用的结构化思考模板和评分标准。

---

## 1. 查询扩展（Query Expansion）

为每个关键词生成搜索变体以提高召回率。Agent 自行完成，不需要外部 AI。

**思考指引：**

对于用户给出的关键词，生成 3-5 个搜索查询变体，覆盖：
- 原词的不同写法（中英文、缩写、全称）
- 核心概念的同义表达
- 去掉版本号的简化形式（如果有版本号）
- 常见的搜索组合词

**示例：**

| 原关键词 | 变体 |
|---------|------|
| GPT-5 | GPT5, GPT-5 发布, GPT-5 release, OpenAI GPT-5 |
| Claude Sonnet 4.6 | Claude 4.6, Claude Sonnet, Anthropic Claude |
| Agentic Coding | 代理编程, 智能体编程, AI Agent Coding, Agent Coding |
| Cursor | Cursor IDE, Cursor AI, Cursor 编程 |

**约束：**
- 不要生成过于宽泛的变体（如将 "Claude Sonnet 4.6" 扩展为 "AI"）
- 变体必须和原关键词指向同一个具体事物
- 3-5 个变体即可，不必过多

---

## 2. 热点相关性分析

对每条搜索结果，按以下结构化模板进行分析判断。

**分析模板：**

针对搜索结果逐条思考以下问题：

```
标题：[结果标题]
摘要：[结果摘要/片段]
来源：[来源名称]
链接：[URL]
监控关键词：[关键词]
查询变体：[使用的变体列表]
```

**判断维度（按顺序）：**

### A. 相关性判定 (relevance_type)

- **direct** — 内容的主题围绕关键词展开。标题或摘要的核心话题就是关键词。
- **indirect** — 关键词不是主题，但内容中有实质性提及或关联。
- **none** — 与关键词无关，或仅在背景中一笔带过。

**判定规则：**
- 同名不同领域的人/事物 → none（如搜 "Sam Altman" 出现 "Breno Altman" 的文章）
- 个人闲聊/段子/碎碎念，即使提到关键词也是 none（除非含有新闻价值的实质信息）
- 仅在背景中一笔带过的提及 → none
- 综合对比文章中实质性讨论了关键词 → indirect

### B. 关键词是否出现 (keyword_mentioned)

- 标题或摘要中是否明确出现了关键词原文或其任一查询变体
- true / false

### C. 真实性验证 (is_verified)

- 内容是否来自可靠来源（官方博客、主流媒体、知名自媒体）
- 是否为真实新闻事件（非标题党/谣言/未证实传闻/恶搞段子）
- true / false

### D. 置信度 (confidence)

- 对整体判断的置信度，0.0 到 1.0
- 信息充分、来源明确 → 0.8-1.0
- 信息有限但基本可判断 → 0.5-0.7
- 难以判断 → 0.3-0.5

### E. 热度评分 (heat_score)

1-10 分，评分标准：

| 分数 | 条件 |
|------|------|
| 9-10 | 重大官方发布/行业里程碑事件（如新模型正式发布、重大收购）|
| 7-8 | 重要更新/有广泛影响的新闻（如产品重大更新、行业政策变化）|
| 5-6 | 有价值的行业动态（如融资新闻、技术突破论文、产品测评）|
| 3-4 | 一般行业资讯（如产品小更新、普通报道）|
| 1-2 | 低价值内容（旧闻重发、翻译转载、个人观点无新信息）|

### F. 中文摘要 (summary)

- 用一句中文概括核心信息，100 字以内
- 如果是英文内容，翻译为中文
- 突出关键事实：谁做了什么、有什么影响

---

## 3. 过滤规则

分析完成后，按以下规则决定保留或丢弃：

```
通过条件（必须全部满足）：
  ✅ relevance_type ≠ none
  ✅ confidence ≥ 0.6
  ✅ heat_score ≥ 3
  ✅ 如果 relevance_type = indirect，则 keyword_mentioned 必须 = true

丢弃条件（命中任一即丢弃）：
  ❌ relevance_type = none
  ❌ confidence < 0.6
  ❌ heat_score < 3
  ❌ indirect 且 keyword_mentioned = false
  ❌ 发布时间超过 10 天
  ❌ 标题/摘要过短（标题 < 10 字 或 摘要 < 20 字）
```

---

## 4. 去重合并

在分析通过后、输出前执行去重：

1. **URL 相同** → 保留热度更高的一条
2. **标题高度相似**（包含关系或仅差异标点/空格）→ 合并，保留信息更完整的版本
3. **同一事件的不同报道** → 保留来源最权威 + 热度最高的一条，摘要中可以提及"多个来源报道"

---

## 5. 结果输出格式

对通过过滤的热点，按热度评分降序排列，输出格式：

```markdown
## 🔥 热点扫描结果 — "{关键词}" ({日期})

发现 {N} 条相关热点：

### 1. [{标题}]({原文链接})
- 📊 热度：{heat_score}/10 | 🏷️ 来源：{source} | 📅 {发布日期}
- 📝 {中文摘要}
- ✅ 真实性：{已验证/待验证} | 相关性：{直接相关/间接相关}

### 2. [{标题}]({原文链接})
...

---
*扫描时间：{当前时间} | 数据源：{使用的源列表}*
```

如果没有发现任何通过过滤的热点，输出：

```markdown
## 📭 热点扫描结果 — "{关键词}" ({日期})

本次扫描未发现高质量相关热点。可能原因：
- 该关键词近期没有重大新闻
- 搜索结果质量未达到过滤门槛

建议：尝试更换关键词或扩大搜索范围。
```
