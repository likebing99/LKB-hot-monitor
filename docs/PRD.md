# LKB Hot Monitor — 产品需求文档 (PRD)

## 1. 项目背景

作为 AI 博主，需要第一时间获取 AI 领域的热点动态（如大模型更新、重大发布等），不能依赖手动搜索。需要一个自动化工具来：
- 自动监控指定关键词，发现真实热点后即时通知
- 定期收集指定领域的热点信息，方便查看
- 利用 AI 识别过滤虚假/标题党内容

## 2. 产品形式

- **Web 应用**：响应式兼容的网页界面（桌面端 + 移动端）
- **Agent Skills**：封装为技能，可交给其他 AI Agent 使用

> 开发顺序：先完成网页版并确保功能正常，再开发 Agent Skills

## 3. 核心功能

### 3.1 关键词监控

| 项目 | 说明 |
|------|------|
| 功能描述 | 用户手动输入要监控的关键词（如 "GPT-5"、"Claude 4"） |
| 触发条件 | 当该关键词相关的**真实内容**出现时 |
| AI 甄别 | 利用 AI 识别假冒/标题党内容，仅推送真实信息 |
| 通知方式 | 第一时间通过 WebSocket / 浏览器推送 / 邮件 通知用户 |
| 检查频率 | 每 30 分钟 |

### 3.2 热点雷达

| 项目 | 说明 |
|------|------|
| 功能描述 | 用户输入监控范围/领域（如 "AI编程"、"大模型"） |
| 自动收集 | 每 30 分钟从多个信息源聚合该领域的热点 |
| AI 分析 | 提取摘要、热度评分（1-10）、去重合并 |
| 展示形式 | 时间线/卡片流，用户可随时查看 |

### 3.3 通知系统

| 通知方式 | 场景 |
|----------|------|
| WebSocket 实时推送 | 页面打开时即时接收 |
| 浏览器 Push 通知 | 页面关闭时桌面通知 |
| 邮件通知（可选） | 用户配置邮箱后，重要热点邮件提醒 |

### 3.4 AI 分析能力

- **真假识别**：判断新闻是否为真实发布/官方确认
- **摘要生成**：为每条热点生成中文简要摘要
- **热度评分**：1-10 分评估热度等级
- **去重合并**：识别同一事件的不同报道并合并

## 4. 数据源

| 数据源 | 方式 | 说明 |
|--------|------|------|
| 网页搜索爬虫 | 后端直接爬取搜索引擎结果页 | 无需 API Key，控制频率 |
| Twitter/X | twitterapi.io API | 获取实时社交媒体信息 |
| RSS 聚合 | 主流科技/AI 媒体 RSS | TechCrunch、The Verge、36Kr 等 |

> 多源交叉验证，避免单一信息源遗漏

## 5. 用户界面要求

- **风格独特**：不要千篇一律，采用极光赛博朋克风
- **响应式**：桌面端侧边栏布局，移动端底部 Tab 切换
- **核心页面**：
  - 关键词管理（添加/删除/启停）
  - 热点时间线（卡片流展示）
  - 通知中心
  - 设置页面（API Key 配置、邮箱配置等）

## 6. Agent Skills（第二阶段）

将核心功能封装为 VS Code Agent Skills：
- `monitor-keyword`：监控关键词并返回命中结果
- `discover-hotspots`：发现指定领域热点

## 7. 非功能需求

- 开发方式：敏捷，不过多工程化，快速上线
- 数据存储：本地 SQLite，零配置
- 爬虫频率控制：避免被封禁
- API Key 管理：通过 .env 文件配置
