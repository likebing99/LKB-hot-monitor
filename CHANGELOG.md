# 更新日志 (CHANGELOG)

## V2.1 — 排序修复与交互体验优化 (2026-03-23)

**相比 V2.0 的变更：**

### Bug 修复

- **扫描按钮闪退修复**：修复 `/api/hotspots/refresh` 路由被 `/:id` 拦截的路由顺序问题；新增所有关键词未启用时的 `no_keywords` 提示，避免扫描无效执行
- **通知数据读取修复**：`getNotifications` 改为手动 `fetch()` 调用，解决 `request()` 自动解包导致 `unreadCount`、`pagination` 丢失的问题
- **通知内容为空修复**：修复 `runSql()` 中 `saveDatabase()` 在 `last_insert_rowid()` 之前执行，导致通知关联的 `hotspot_id` 始终为 0 的问题；后端过滤 `hotspot_id > 0` 的记录，前端通知卡片显示关联热点标题、摘要、热度与原文链接
- **发布时间排序修复**：修复 Twitter 格式日期（`Sun Mar 15...`）按字母排序而非时间排序的问题；入库时统一转为 ISO 格式，启动时自动迁移历史非 ISO 日期数据

### 仪表盘交互优化

- **排名标记**：热点卡片左侧数字改为显示当前排序下的排名（#1, #2, #3...），前三名高亮
- **热度标签**：每张热点卡片标签行新增「热度 X」标签（橙色样式），与筛选栏热度范围筛选对应，方便用户理解热度评分
- **排序文案调整**：「热度综合」排序按钮更名为「互动数据」，避免与 AI 热度评分概念混淆；该排序依据 `likes + retweets×3 + views×0.001` 计算
- **无关键词扫描提示**：当所有关键词均未启用时，点击扫描按钮会显示警告提示

### 修改文件清单

| 文件 | 变更说明 |
|------|---------|
| `client/src/pages/Dashboard.jsx` | ScoreBadge 改为排名显示；新增热度标签；排序按钮「热度综合」→「互动数据」；新增无关键词扫描警告 |
| `client/src/pages/Notifications.jsx` | 通知卡片可点击，显示热点标题/摘要/热度/原文链接 |
| `client/src/components/Layout.jsx` | 通知未读数改用 `unreadCount` 字段 |

## V2.0 — 多源扩展与稳定性增强 (2026-03-23)

**相比 V1.1 的变更：**

### 热点采集能力升级

- **Web 数据源重构**：重构 Web 抓取链路，采用 Bing、Google News RSS、DuckDuckGo、Brave News、B站、微博 多源组合采集
- **B站接入**：支持按关键词搜索视频，也支持优先识别 UP 主并抓取其最新视频
- **微博接入**：接入微博移动端公开搜索接口，支持按用户和关键词抓取微博内容
- **twitterapi.io 检索增强**：改为高级搜索，排除回复与引用推文，仅保留原创内容

### 热点过滤与数据质量治理

- **新鲜度过滤加固**：仅保留发布时间在 10 天以内的热点；无时间戳或时间不可解析的结果直接丢弃
- **推文质量过滤**：对 twitterapi.io 返回的数据增加互动量、浏览量、文本长度等硬门槛过滤
- **URL 去重增强**：在标题去重基础上新增 `source_url` 去重，减少重复热点入库
- **AI 分析提速**：热点分析改为 5 并发批处理，并支持逐条分析完成后实时入库与推送
- **AI 超时保护**：OpenRouter 请求新增超时中断，避免单次调用拖慢整轮扫描

### 调度与清理机制升级

- **扫描超时保护**：扫描任务新增 180 秒超时释放锁，避免任务卡死
- **每日自动清理**：新增北京时间每天 `00:00:01` 清理任务，自动删除发布时间超过 10 天的旧热点及关联通知
- **启动时清理**：后端启动时立即执行一次旧热点清理，保证数据新鲜度
- **手动清理接口**：新增 `/api/cleanup` 接口，可手动触发旧数据清理
- **默认扫描周期调整**：后端默认扫描间隔由 30 分钟调整为 180 分钟

### 前端交互与仪表盘优化

- **外链按钮修复**：修复热点卡片“查看原文”按钮被遮挡、无法点击的问题
- **扫描倒计时**：新增扫描中倒计时与自动恢复状态，优化手动扫描反馈
- **实时推送优化**：WebSocket 新热点推送按当前筛选条件实时更新列表与统计
- **筛选逻辑调整**：仪表盘默认仅展示已验证热点，不再显示“仅已验证”复选框
- **搜索体验优化**：搜索框文案改为“在已抓取的热点中筛选...”，避免误解为全网检索
- **来源筛选扩展**：仪表盘来源筛选新增 B站、微博，并补充来源标签样式

### 统计与运行稳定性修复

- **今日热点统计修复**：改为按北京时间计算今日热点，修复 UTC 时区导致的统计异常
- **环境变量加载修复**：后端改为按 `server/.env` 绝对路径加载配置，不再依赖启动目录
- **twitterapi.io 凭证回退**：扫描器支持优先读环境变量，并回退读取数据库中的 `twitter_api_key`
- **后端任务修复**：更新 VS Code Backend 任务启动方式，避免因终端命令失败导致持续报错

### 数据治理结果

- **旧数据清理**：按 `published_at` 实际发布时间清理超过 10 天的旧热点
- **时间格式验证**：确认当前数据库热点发布时间均可被 JavaScript 正确解析
